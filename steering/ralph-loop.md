# Ralph 루프

작업 완료까지 자동 검증 및 반복.

## 프로세스

```
실행 → 테스트 → 리뷰 → 문제? → 수정 → 반복 → 완료
```

## 단계별 서브에이전트

### 1. 실행 (Implementation)
- **executor**: 코드 구현, 파일 수정

### 2. 테스트 (Testing)
- **tester**: 테스트 실행, 커버리지 확인
- 테스트 실패 시 → debugger 호출

### 3. 리뷰 (Review)
- **reviewer**: 코드 품질, 베스트 프랙티스 확인
- 문제 발견 시 → executor 재호출

### 4. 수정 (Fix)
- **debugger**: 버그 수정, 에러 해결
- 수정 후 → 2단계(테스트)로 돌아감

### 5. 완료 (Complete)
- 모든 테스트 통과 ✓
- 리뷰 승인 ✓
- 요구사항 충족 ✓

## 워크플로우 예시

### 예시 1: 새 기능 구현

```
1. executor: 로그인 폼 구현
2. tester: 테스트 실행 → 2개 실패
3. debugger: 실패 원인 수정
4. tester: 테스트 재실행 → 모두 통과 ✓
5. reviewer: 코드 리뷰 → 보안 이슈 발견
6. executor: 보안 이슈 수정
7. tester: 테스트 재실행 → 통과 ✓
8. reviewer: 최종 승인 ✓
9. 완료
```

### 예시 2: 버그 수정

```
1. debugger: 버그 분석 및 수정
2. tester: 회귀 테스트 → 통과 ✓
3. reviewer: 수정 검토 → 승인 ✓
4. 완료
```

## 반복 규칙

- **최대 5회 반복** - 해결 안 되면 사용자에게 보고
- **자동 위임** - 각 단계마다 적절한 서브에이전트 호출
- **빠른 피드백** - 각 단계 결과 즉시 확인

## 서브에이전트 선택

| 상황 | 호출할 에이전트 |
|------|----------------|
| 코드 작성/수정 | executor |
| 테스트 실행/작성 | tester |
| 버그 수정 | debugger |
| 코드 리뷰 | reviewer |
| 아키텍처 검토 | architect |

## 중요

- **모든 단계 필수** - 실행 → 테스트 → 리뷰
- **자동 반복** - 문제 발견 시 자동으로 수정 단계
- **명확한 피드백** - 각 단계 결과 명시
- **중단 시점** - 5회 반복 후 사용자 개입

# Oh My Kiro - 기본 원칙

## 언어 규칙

- **모든 응답은 한국어로 작성**
- 기술 용어는 한글과 영문 병기 (예: 오케스트레이션(Orchestration))
- 코드 주석은 한글로 작성
- 변수명/함수명은 영문 camelCase 유지
- 커밋 메시지는 한글로 작성

## 핵심 원칙

1. **직접 해결 우선** - 가능하면 작업을 직접 해결
2. **필요시 위임** - 품질이나 속도를 향상시킬 때만 위임
3. **구체적 진행** - 진행 상황을 구체적이고 검증 가능하게
4. **증거 기반** - 가정보다 증거 사용
5. **최소 경로** - 품질을 유지하는 가장 가벼운 경로

## 작업 규칙

### 코드 작성
- 클린하고 유지보수 가능한 코드
- 기존 패턴 따르기
- 새 코드에 테스트 추가
- 변경사항 최소화

### 문서 작성
- 명확하고 구조화된 문서
- 예제 포함
- 한글 우선, 필요시 영문 병기

### 에러 처리
- 체계적 디버깅
- 근본 원인 분석
- 재현 단계 문서화
- 수정 후 검증

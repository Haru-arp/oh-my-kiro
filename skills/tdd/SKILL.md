# TDD

테스트 주도 개발 워크플로우

## 사용법

```
$tdd "사용자 인증 구현"
$tdd "결제 처리 추가"
```

## 워크플로우

1. 실패하는 테스트를 먼저 작성
2. 통과하는 최소한의 코드 구현
3. 테스트를 유지하면서 리팩토링
4. 커버리지 검증

## 규칙

- **모든 응답은 한국어로 작성**
- 항상 테스트를 먼저 작성
- 테스트가 실패하는지 확인
- 최소한의 코드로 통과시키기
- 리팩토링 후 테스트 재실행
